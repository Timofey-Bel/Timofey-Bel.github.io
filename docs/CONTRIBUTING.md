# 📝 Руководство по документации MedX

## Структура документации

Вся документация проекта находится в папке `docs/` и организована следующим образом:

### Основные файлы

```
docs/
├── README.md                 # Главная страница документации
├── DOCS_INDEX.md            # Полная навигация по документам
├── QUICK_START.md           # Быстрый старт (5 минут)
├── README_PHP85.md          # Полное руководство
├── SUMMARY.md               # Краткое резюме проекта
├── MIGRATION_README.md      # Детали миграции из legacy
├── OSPANEL_SETUP.md         # Настройка OSPanel
├── PHP_8.5_UPGRADE.md       # Обновление PHP до 8.5
├── CHANGELOG_PHP85.md       # История изменений PHP
└── CONTRIBUTING.md          # Этот файл
```

## Правила создания документации

### 1. Расположение файлов

✅ **Все новые документы создавать в папке `docs/`**

```bash
# Правильно
docs/NEW_FEATURE.md

# Неправильно
NEW_FEATURE.md
```

### 2. Формат файлов

- Используйте Markdown (.md)
- Кодировка: UTF-8
- Переносы строк: LF (Unix)

### 3. Структура документа

Каждый документ должен содержать:

```markdown
# Заголовок документа

Краткое описание (1-2 предложения)

## Содержание (опционально)

## Основные разделы

### Подразделы

---

**Последнее обновление:** Дата
**Версия:** X.X.X
```

### 4. Ссылки между документами

Используйте относительные ссылки:

```markdown
# Правильно
[QUICK_START.md](QUICK_START.md)
[Главная](../README.md)

# Неправильно
[QUICK_START.md](/docs/QUICK_START.md)
[Главная](../../README.md)
```

### 5. Именование файлов

- Используйте UPPERCASE для основных документов: `README.md`, `SUMMARY.md`
- Используйте snake_case для специфичных: `feature_name.md`
- Используйте понятные имена: `PHP_8.5_UPGRADE.md` вместо `php_upgrade.md`

## Типы документов

### Руководства (Guides)
- Пошаговые инструкции
- Примеры: `QUICK_START.md`, `OSPANEL_SETUP.md`

### Справочники (References)
- Полная информация о функционале
- Примеры: `README_PHP85.md`, `MIGRATION_README.md`

### Обзоры (Overviews)
- Краткая информация
- Примеры: `SUMMARY.md`, `DOCS_INDEX.md`

### История (Changelogs)
- Изменения и обновления
- Примеры: `CHANGELOG_PHP85.md`

## Обновление документации

### При добавлении нового документа:

1. Создайте файл в папке `docs/`
2. Обновите `docs/DOCS_INDEX.md`
3. Обновите `docs/README.md` (если нужно)
4. Добавьте ссылку в главный `README.md` (если нужно)

### Пример:

```bash
# 1. Создаем новый документ
touch docs/NEW_FEATURE.md

# 2. Редактируем DOCS_INDEX.md
# Добавляем ссылку на новый документ

# 3. Коммитим изменения
git add docs/
git commit -m "docs: добавлена документация NEW_FEATURE"
```

## Стиль написания

### Тон
- Дружелюбный и понятный
- Избегайте жаргона (или объясняйте его)
- Пишите для разных уровней опыта

### Форматирование

```markdown
# Заголовки
Используйте иерархию: #, ##, ###

# Код
Используйте блоки кода с указанием языка:
```bash
php artisan serve
```

# Списки
- Маркированные для неупорядоченных
1. Нумерованные для последовательных шагов

# Выделение
**Жирный** для важного
*Курсив* для акцента
`Код` для команд и переменных

# Ссылки
[Текст ссылки](URL)

# Изображения (если нужны)
![Alt текст](путь/к/изображению.png)
```
